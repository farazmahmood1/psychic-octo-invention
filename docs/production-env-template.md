# OpenClaw Production Environment Variables

Complete reference for deploying OpenClaw to Render.com (or any cloud provider).

---

## Required Variables

### Core Infrastructure

| Variable | Service | Source | Example |
|----------|---------|--------|---------|
| `DATABASE_URL` | API, Worker | **Auto-injected by Render** (from Render Postgres) | `postgresql://user:pass@host:5432/dbname` |
| `REDIS_URL` | API, Worker | **Auto-injected by Render** (from Render Redis) | `redis://red-xxxx:6379` |
| `SESSION_SECRET` | API | **Auto-generated by Render** (`generateValue: true`) | 64+ random characters |
| `NODE_ENV` | API, Worker | Set manually | `production` |
| `PORT` | API | Set manually (Render default) | `4000` |

### AI / Model Routing

| Variable | Service | Source | Example |
|----------|---------|--------|---------|
| `OPENROUTER_API_KEY` | API, Worker | **Must be set manually** | `sk-or-v1-xxxxxxxxxxxx` |
| `OPENROUTER_BASE_URL` | API, Worker | Has default, no change needed | `https://openrouter.ai/api/v1` |

### App URLs

| Variable | Service | Source | Example |
|----------|---------|--------|---------|
| `APP_BASE_URL` | API | **Must be set manually** to the Render API service URL | `https://openclaw-api.onrender.com` |
| `ADMIN_APP_URL` | API | **Must be set manually** to the Render static site URL | `https://openclaw-admin.onrender.com` |
| `API_BASE_URL` | API | **Must be set manually** (should match `APP_BASE_URL`) | `https://openclaw-api.onrender.com` |
| `RENDER_EXTERNAL_URL` | API | **Auto-injected by Render** for web services | `https://openclaw-api.onrender.com` |

> **IMPORTANT:** `APP_BASE_URL`, `ADMIN_APP_URL`, and `API_BASE_URL` must NOT be `localhost` in production.
> - `APP_BASE_URL` = your Render API service's public URL
> - `ADMIN_APP_URL` = your Render static site's public URL (used for CORS)
> - `API_BASE_URL` = same as `APP_BASE_URL`
> - Provider webhooks such as Resend inbound email must target `API_BASE_URL`, not the marketing/root site if that root domain serves static HTML

## Integration Variables (Optional — system runs in degraded mode without these)

### Telegram Bot

| Variable | Service | Source | Example |
|----------|---------|--------|---------|
| `TELEGRAM_BOT_TOKEN` | API, Worker | **Must be set manually** (from @BotFather) | `123456789:AABBccDDeeFFggHH` |
| `TELEGRAM_WEBHOOK_SECRET` | API | **Auto-generated by Render** (`generateValue: true`) | Min 16 characters |
| `TELEGRAM_API_BASE_URL` | API | Has default, no change needed | `https://api.telegram.org` |

> The API auto-registers the Telegram webhook on startup using `APP_BASE_URL` or `RENDER_EXTERNAL_URL`.

### Email (SMTP)

| Variable | Service | Source | Example |
|----------|---------|--------|---------|
| `SMTP_HOST` | API, Worker | **Must be set manually** | `smtp.gmail.com` |
| `SMTP_PORT` | API, Worker | Has default | `587` |
| `SMTP_USER` | API, Worker | **Must be set manually** | `noreply@yourdomain.com` |
| `SMTP_PASS` | API, Worker | **Must be set manually** | App-specific password |
| `SMTP_FROM` | API, Worker | **Must be set manually** | `OpenClaw <noreply@yourdomain.com>` |
| `INBOUND_EMAIL_WEBHOOK_SECRET` | API | **Auto-generated by Render** (`generateValue: true`) | Min 16 characters |

### GoHighLevel CRM

| Variable | Service | Source | Example |
|----------|---------|--------|---------|
| `GHL_API_BASE_URL` | API, Worker | Has default, no change needed | `https://rest.gohighlevel.com/v1` |
| `GHL_API_TOKEN` | API, Worker | **Must be set manually** | `pit-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` |

### Google Sheets (Bookkeeping)

| Variable | Service | Source | Example |
|----------|---------|--------|---------|
| `GOOGLE_SERVICE_ACCOUNT_JSON` | API, Worker | **Must be set manually** | See below |
| `GOOGLE_SHEETS_BOOKKEEPING_SPREADSHEET_ID` | API, Worker | **Must be set manually** | `1jCzBLc2oCt3uLmwr8R0NJHP9BR_1TFYid2ZY2AfZ5TY` |

> **CRITICAL:** `GOOGLE_SERVICE_ACCOUNT_JSON` must contain the **complete** JSON from Google Cloud Console.
> It MUST include these fields (at minimum):
> ```json
> {
>   "type": "service_account",
>   "project_id": "your-project-id",
>   "private_key_id": "key-id",
>   "private_key": "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n",
>   "client_email": "your-sa@your-project.iam.gserviceaccount.com",
>   "client_id": "123456789",
>   "auth_uri": "https://accounts.google.com/o/oauth2/auth",
>   "token_uri": "https://oauth2.googleapis.com/token"
> }
> ```
> Missing `private_key` or `client_email` will cause Google Sheets integration to fail silently.

## Optional Tuning Variables

| Variable | Service | Default | Description |
|----------|---------|---------|-------------|
| `WORKER_CONCURRENCY` | Worker | `5` | Max concurrent BullMQ job processors (1-50) |
| `REQUEST_TIMEOUT_MS` | API | `30000` | HTTP request timeout in milliseconds |
| `MAX_PAYLOAD_SIZE` | API | `2mb` | Max request body size |

## Admin Seed (Run Once After First Deploy)

| Variable | Service | Source | Description |
|----------|---------|--------|-------------|
| `ADMIN_SEED_EMAIL` | API | **Must be set manually** | Email for the initial admin user |
| `ADMIN_SEED_PASSWORD` | API | **Must be set manually** | Password (min 12 chars, must include uppercase, lowercase, number, symbol) |

> Run `npm run seed:admin` after first deploy to create the initial admin user.
> These variables can be removed after seeding.

## Pre-Deploy Checklist

- [ ] `NODE_ENV` set to `production`
- [ ] `APP_BASE_URL` set to Render API service URL (not localhost)
- [ ] `ADMIN_APP_URL` set to Render static site URL (not localhost)
- [ ] `API_BASE_URL` matches `APP_BASE_URL`
- [ ] `OPENROUTER_API_KEY` is a valid OpenRouter API key
- [ ] `GOOGLE_SERVICE_ACCOUNT_JSON` contains the FULL JSON (with `private_key` and `client_email`)
- [ ] Google Sheets spreadsheet shared with the service account's `client_email`
- [ ] `TELEGRAM_BOT_TOKEN` is a valid token from @BotFather
- [ ] `GHL_API_TOKEN` is a valid GoHighLevel API token
- [ ] SMTP credentials tested (send a test email before deploying)
- [ ] Database migrations run (`npx prisma migrate deploy --schema=prisma/schema.prisma`)
- [ ] Admin seed user created (`npm run seed:admin`)
- [ ] Health check endpoint responds: `GET /health`
